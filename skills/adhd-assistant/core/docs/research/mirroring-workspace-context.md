# Implementation Guide: Mirroring Workspace Context

## 🎯 Objective
Combat "Task Blindness" and "Out of Sight, Out of Mind" symptoms in ADHD users by projecting the agent's internal focus into the user's physical workspace.

## 🔍 Research Origin
- **Source**: `anthropics/task-management` (Knowledge Work Plugins)
- **Concept**: Shared Workspace State (`TASKS.md`).
- **Adaptation**: Instead of a full list (which causes overwhelm), we will project only the **Current Focus** and immediate **Next Steps**.

## 🛠 Technical Design

### Trigger
Every time `engine/state_store.py:save()` is called and the `execution.current_front` has changed.

### Output
A file named `CURRENT_FOCUS.md` in the repository root.

### Content Template
```markdown
# 🎯 Current Focus
**Task:** {{state.execution.current_front}}

## ⚡ Next Micro-steps
- [ ] {{state.execution.next_actions[0]}}
- [ ] {{state.execution.next_actions[1]}}

---
*Generated by ADHD Assistant*
```

## 🧠 ADHD Cognitive Value
Acts as **External Working Memory**. By placing the file in the project root, it appears in `ls`, file explorers, and IDE sidebars, providing a "nudge" every time the user interacts with the terminal or code.
